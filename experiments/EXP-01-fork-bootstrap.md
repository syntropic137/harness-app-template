## Mission slice
Pressure-test the keystone adopter promise: a fresh clone + `just bootstrap` succeeds,
and a first trivial commit cleanly passes the APSS doc gate (no "APSS binary not found",
no missing-ADR refusal on an unrelated change).

## Frozen hypothesis
1. `just bootstrap` exits 0 on the existing working tree without operator-only steps
   (no manual ADR creation, no env file editing).
2. The doc-validator commit hook is enabled and runs the APSS gate; commits succeed
   when the change does not touch the ADR or principle surfaces it polices.
3. The hook resolves the APSS binary deterministically (vendored shim, PATH entry, or
   wrapper produced by bootstrap) and does NOT bail out with "APSS binary not found"
   on the first commit a new clone produces.

## Frozen prediction
- 1 will PASS: `just bootstrap` exits 0.
- 2 will PASS for a benign change (adding a new file under experiments/ that does not
  reference an ADR or principle).
- 3 will PASS: the APSS binary resolves via either the vendored cargo workspace
  (recent commit `fix(apss): exclude .apss/build from cargo workspace` plus
  `ci(docs): compose project APSS CLI before doc validation`) or a wrapper
  surfaced through `just bootstrap`.

Combined verdict prediction: CONFIRMED (all three).
Risk of FALSIFIED: hook is registered but APSS binary not yet built locally;
bootstrap does not produce the binary itself.
