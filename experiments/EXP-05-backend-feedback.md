## Mission slice
Hit the API directly, read structured responses; verify there is a DOCUMENTED
path from "I am a fresh agent" to "I know what endpoints exist and how to

## Frozen hypothesis
1. The bare template ships NO application-level HTTP API. All three
   `ws_apps/example-*` are one-shot emit-and-exit telemetry samples; their
   only "endpoints" are their telemetry exports.
2. The only API surface a fresh agent CAN hit directly today is the
   observability stack: VictoriaTraces (Jaeger-shaped), VictoriaLogs (LogsQL),
   and the OTEL Collector OTLP receivers.
3. That observability API surface IS documented for agents via the
   `observability-queries` skill in `.claude/skills/`, NOT via AGENTS.md
   itself. AGENTS.md lists the skill by name but the actual endpoint
   surface lives only inside the skill body.
4. There is NO Swagger / OpenAPI / introspection endpoint for the
   observability stack itself; agents discover the API shape only by
   reading the skill body or the upstream VictoriaMetrics docs.

## Frozen prediction
- 1 PASS: zero HTTP server frameworks in any example app's deps.
- 2 PASS: observability stack ports respond.
- 3 PASS: AGENTS.md gives only a skill pointer; skill body has the curl
  examples.
- 4 PASS: no OpenAPI introspection ships in the template.

Combined verdict prediction: CONFIRMED. The "backend API" axis is gated on
the agent INVOKING the observability-queries skill, not on AGENTS.md alone.
