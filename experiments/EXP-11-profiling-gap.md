## Mission slice
Can an agent profile API latency and frontend performance with what ships in
the harness today? Prediction (per roadmap): NO. This experiment FEEDS the
profiling-slot design task; the deliverable is "exactly what is missing".

## Frozen hypothesis
1. There is NO profiling slot in the 11 named slots
   (`stack-manager`, `inspector`, `hooks`, `telemetry-sdk`,
   `observability-stack`, `sensors`, `agent-plugins`, `task-runner`,
   `secret-scanner`, `doc-validator`, `versioning`); AGENTS.md does not
   mention profiling.
2. There is NO `just profile` recipe, no `pprof`-equivalent for any of the
   example apps, and no CDP Performance trace harness wired in.
3. Latency histograms are NOT computed by the observability stack today
   for the example apps (no `http.server.duration` style histograms because
   no HTTP server runs).
4. Frontend perf profiling: the `chrome-devtools-deep` skill EXISTS and
   covers Performance traces conceptually, but the template does not ship
   a runnable script that boots the docs UI, captures a trace, saves an
   artifact, and correlates it with a backend trace ID.
5. The only flame-graph-shaped artifact an agent could PRODUCE today is by
   manually using Chrome DevTools through the `playwright-debug` /
   `chrome-devtools-deep` skill recipes. There is no `just inspector record`
   or `just profile` shortcut.

## Frozen prediction
- 1 PASS: no profiling slot.
- 2 PASS: no `just profile`.
- 3 PASS: no app-emitted histograms in VictoriaMetrics.
- 4 PASS: no boot-trace-correlate harness.
- 5 PASS: no flamegraph shortcut.

Combined verdict prediction: CONFIRMED. The "what is missing" list is the
deliverable.

## Probes
- `grep -nE "profil|profile|flamegraph|pprof|perf trace|chrome.*trace|cpu.*trace" AGENTS.md README.md justfile docs/standard/v0.1.md 2>/dev/null` for documented intent.
- `just --list` and `grep -nE "profile|trace.*capture|flame" justfile` for recipe presence.
- `find harness -type d -maxdepth 2` to enumerate the actual slot layout
  and confirm no profiling slot exists.
- `curl http://localhost:${VM_PORT}/api/v1/label/__name__/values` to list all
  metric names; grep for `duration`, `latency`, `histogram` to score 3.
- `ls .claude/skills/ | grep -i profil` to test for a profiling skill.
- Read `chrome-devtools-deep/SKILL.md` and check whether it provides a
  one-shot recipe for "boot UI -> capture trace -> save -> diff" or only
  conceptual guidance.
