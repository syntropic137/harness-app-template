## Mission slice
Give yourself a small real feature: add a backend endpoint plus a UI
element that calls it. Self-serve the WHOLE loop in this harness: code
the change, run the apps, observe both backend AND UI feedback, query
telemetry to confirm the call, get the gates green. Score WHERE the
harness carried you vs WHERE it left you stuck.

The feature is intentionally tiny so the loop, not the feature, is the
experimental surface.

## Feature
Add a Next.js Route Handler at `ws_apps/docs/app/api/hello/route.ts` that
returns `{ "hello": "<name>", "trace_id": "<id>", "ts": <unix_ms> }`.
Add a button on the docs home page that fetches the endpoint and renders
the response inline.

This lives entirely inside `ws_apps/docs` (Next.js gives us both
endpoint AND UI in one process) so HMR can show both halves of the loop
without spinning up a separate backend.

## Frozen hypothesis
1. The Next.js Route Handler convention works out of the box: editing
   `ws_apps/docs/app/api/hello/route.ts` and curling
   `http://localhost:3001/api/hello` returns JSON without restarting the
   dev server.
2. Adding a client component button to `ws_apps/docs/app/page.tsx` will
   show the response inline after one HMR cycle (~10-20s including
   compile, per EXP-03 measurements).
3. Telemetry from the Route Handler call WILL NOT reach VictoriaTraces
   automatically. The docs app does not import `@harness/telemetry`,
   has no OTEL instrumentation, and there is no documented "wire next.js
   to OTEL" recipe in the template.
4. The fitness / sensors gate WILL be sensitive to my new file: a new
   API route under `ws_apps/docs/app/api/` may change the depcruise
   coupling values for `ws_apps/docs/app/` and possibly fail the gate
   without a baseline update.
5. The APSS doc-validator gate WILL still block ALL commits (per EXP-01
   finding); my feature commits will need `--no-verify`.

## Frozen prediction
- 1 PASS: Route Handler hot-reloads and returns JSON.
- 2 PASS: UI shows response after HMR.
- 3 PASS (sad): no auto telemetry for the route handler.
- 4 PASS (sad): sensors gate fires on the new file structure.
- 5 PASS (sad): APSS gate still blocks; --no-verify still required.

Combined verdict prediction: PARTIAL. The DEV LOOP for code-run-observe
carries the agent through cleanly inside Next.js; everything OUTSIDE
the Next.js dev loop (telemetry wiring, doc gate, sensors gate)
leaves the agent stuck.
