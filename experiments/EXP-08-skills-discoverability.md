# EXP-08 skills discoverability

Claim: CobaltCoast (claude-opus-4-7), 2026-06-10, branch feat/apss-integration.
Sole-editor; do not edit this file from other agents.

## Mission slice
Which installed skills exist in `.claude/skills/`? Do the relevant ones fire
for the obvious harness tasks (observability lookup, evidence bundle,
experiment scaffolding)? Are any obviously missing relative to the workflow
this template invites?

## Frozen hypothesis
1. Exactly three project-local skills ship in `.claude/skills/`:
   `running-experiments`, `observability-queries`, `before-after-evidence`.
   AGENTS.md lines 29-32 enumerate them; the file system count will match.
2. Each skill has a SKILL.md with non-empty frontmatter (name + description).
3. There is NO skill for "run the app" / "boot the dev loop" / "list the
   running services". The closest match is `observability-queries` and that
   one is query-only.
4. There is NO skill that maps endpoint names to ports (the VL_PORT /
   VM_PORT / VT_PORT trio); the env-var dance lives inline in
   `observability-queries` SKILL.md and in each example app README.
5. The skill bodies refer (by link) to the upstream harness-engineering
   plugin skills documented in AGENTS.md lines 40-95, but those bodies are
   NOT vendored; an agent on a fresh clone sees pointers, not implementations.

## Frozen prediction
- 1 PASS: count = 3.
- 2 PASS: all three have valid frontmatter.
- 3 PASS: no run-the-app skill.
- 4 PASS: ports surface lives inline.
- 5 PASS: upstream plugin skills are link-only on a fresh clone.

Combined verdict prediction: CONFIRMED. The skill surface is small and
correctly enumerated; the gap is a missing "start-here / run-the-app"
skill, which makes EXP-02's discoverability problem worse.

## Probes
- `find .claude/skills -maxdepth 2 -name SKILL.md` to count + locate.
- `head -10 .claude/skills/*/SKILL.md` to inspect frontmatter.
- `grep -ri "run.*app\|dev.*server\|start.*server" .claude/skills/` to find
  any startup-flavored skill.
- `grep -nE "VL_PORT|VM_PORT|VT_PORT" .claude/skills/*/SKILL.md` to confirm
  the port surface lives inline.
- `ls ~/.claude/plugins/harness-engineering/skills/ 2>&1` to test whether
  the upstream plugin is actually installed locally; if not, the AGENTS.md
  reference is link-only for the agent invoking it now.
